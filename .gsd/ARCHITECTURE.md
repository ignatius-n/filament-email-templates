## Overview

The **Filament Email Templates Designer** is a Laravel package that serves as a plugin for Filament v4 and v5. It integrates the visual drag-and-drop editor **Unlayer** directly into Filament's form schema, allowing administrators to design and manage responsive, pixel-perfect HTML email templates from the Filament panel. Developers can then load these templates by a unique programmatic key and send them using standard Laravel Mailables via the `HasEmailTemplate` trait.

## Data Flow

1. **Template Creation/Editing**:
   - The user opens the Filament panel. The `UnlayerEditor` loads the Unlayer JS SDK.
   - For an existing record, the `unlayer_state` (JSON string containing both `design` and `html`) is loaded from the Eloquent model's attributes, and Alpine calls `instance.loadDesign(design)`.
   - As the user changes elements, `design:updated` events trigger, which periodically sets the Livewire state.
   - When the user clicks **Save** or **Create**, JavaScript intercepts the submit button click, exports the latest HTML and JSON from Unlayer asynchronously via `exportHtml()`, and invokes `syncUnlayerExport` on the Livewire page component.
   - The component mutations save `body_html` and `body_json` to the database, completing the action in a single network request to avoid state synchronization race conditions.

2. **Email Dispatch**:
   - The developer triggers a Laravel Mailable class using the `HasEmailTemplate` trait.
   - The Mailable calls `build()`.
   - The trait reads `templateKey` and queries the database for the matching `EmailTemplate`.
   - It parses/renders the subject line via Blade with the provided variables.
   - It performs variable replacements on placeholders in the HTML body (protecting template styling and matching both raw and URL-encoded variable patterns).
   - Finally, it wraps the content inside standard `<html>` elements if missing, and outputs the mailable subject and HTML body.

## Technical Debt

- [ ] **No Automated Tests:** The repository includes a `phpunit.xml` configuration referring to a `tests` directory, but no actual test files or directory exist.
- [ ] **Coupling with Livewire internals (`$wire.call` / `$wire.set`):** The Javascript embed directly calls `syncUnlayerExport` on page classes, coupling it to the specific Filament page implementations.
- [ ] **Manual Blade variable mapping implementation:** The variable replacement regex in `HasEmailTemplate.php` handles basic substitutions but does not fully support advanced Blade template control structures (if statements, loops) inside the HTML body.
